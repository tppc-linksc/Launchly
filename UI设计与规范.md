## 1. 设计原则

### 1.1 产品定位反推的设计语言

Launchly 是**工具型产品**，给开发者和测试人员用。设计语言应：

- **严肃克制**：不堆装饰，不用情绪化色彩
- **数据密度高**：开发者愿意接受较高的信息密度，不是消费 app
- **中性 + 单一主色**：背景中性色（白 / 灰），强调色用一个品牌色
- **快速可读**：状态颜色一眼能识别（绿成功 / 红失败 / 黄进行中）
- **可访问**：色弱友好，关键状态用颜色 + 文字 + icon 三重表达
- **零情绪化弹窗**：不用 toast 庆祝部署成功，用安静的状态卡片
- **键盘友好**：常用操作支持快捷键

### 1.2 参考产品（不抄袭，借鉴语言）

| 产品 | 借鉴的方面 |
| --- | --- |
| [Linear](https://linear.app) | 排版的克制、键盘快捷键体系 |
| [Railway](https://railway.app) | 部署日志卡片、空态文案 |
| [Vercel Dashboard](https://vercel.com) | 项目列表的卡片密度、部署详情布局 |
| [GitHub](https://github.com) | 表格 / 列表 / 状态 badge 的样式 |
| [Plane](https://plane.so) | 中性配色 + 单一主色的克制做法 |

### 1.3 反原则（绝不做）

- 不用渐变色作为主背景或主色
- 不用斜体（开发者审美不接受）
- 不用过度阴影、玻璃拟态、3D 装饰
- 不在工具型页面上放大插画 hero
- 不用动画把"加载成功"做成 confetti
- 不用全屏 modal 替代单独页面

## 2. 视觉规范

### 2.1 色板

#### 2.1.1 主色（Brand）

Launchly 主色：**深青蓝 #1F6FEB**（GitHub 同款蓝，亲切且严肃）

```text
brand-50   #E9F2FE   ← 浅底
brand-100  #C8DFFB
brand-200  #97C0F8
brand-300  #5EA0F4
brand-400  #3F87F0
brand-500  #1F6FEB   ← 主色（按钮 primary、链接、激活态）
brand-600  #1858C0   ← hover
brand-700  #114391   ← pressed
brand-800  #0B2F65
brand-900  #061B3B
```

#### 2.1.2 中性色（Neutral）

```text
neutral-0    #FFFFFF   ← 卡片背景
neutral-25   #FAFBFC   ← 页面背景（亮色模式）
neutral-50   #F6F8FA
neutral-100  #EAEEF2   ← 分割线、表头背景
neutral-200  #D0D7DE   ← 边框
neutral-300  #AFB8C1   ← 禁用文字
neutral-400  #8C959F   ← 占位文字
neutral-500  #6E7781   ← 次要文字
neutral-600  #57606A
neutral-700  #424A53   ← 正文
neutral-800  #32383F
neutral-900  #1F2328   ← 标题
neutral-950  #0D1117   ← 极强调（暗色模式背景基础）
```

#### 2.1.3 状态色

```text
success-50   #DAFBE1   success-500  #1A7F37   success-700  #116329
warning-50   #FFF8C5   warning-500  #D29922   warning-700  #9A6700
danger-50    #FFEBE9   danger-500   #CF222E   danger-700   #82071E
info-50      #DDF4FF   info-500     #0969DA   info-700     #0550AE
```

**使用规则**：
- 部署成功 / 健康检查通过 → success
- 部署进行中 / 等待门禁 → warning
- 部署失败 / SSH 断连 → danger
- 中性消息 / 默认 → info / neutral
- 普通文字提示**不准**用 success 绿（保留语义专属）

#### 2.1.4 暗色模式（Dark）

```text
dark-bg          #0D1117   ← 页面底
dark-surface     #161B22   ← 卡片
dark-surface-2   #1C2128   ← 卡片内嵌
dark-border      #30363D
dark-text-1      #E6EDF3   ← 主文字
dark-text-2      #8D96A0   ← 次文字
dark-text-3      #6E7781   ← 占位
```

亮 / 暗切换通过 `<html data-theme="light|dark">` 控制，所有色板用 CSS 变量。

### 2.2 字体

```text
font-family-sans: -apple-system, BlinkMacSystemFont, "Segoe UI", "PingFang SC",
                  "Hiragino Sans GB", "Microsoft YaHei", "Source Han Sans CN",
                  sans-serif;
font-family-mono: "JetBrains Mono", "Fira Code", Consolas, "Liberation Mono",
                  Menlo, monospace;
```

中文用系统默认（苹方 / 微软雅黑），不准引入 Google Fonts 等外部字体（中国大陆访问问题）。代码块和日志用等宽字体。

### 2.3 字号阶梯

```text
text-xs       12px / 16px   ← 表格次要信息、tag、help text
text-sm       13px / 20px   ← 表格主信息、按钮、次级标题
text-base     14px / 22px   ← 正文（默认）
text-md       16px / 24px   ← 页面标题、card 标题
text-lg       20px / 28px   ← 区块大标题
text-xl       24px / 32px   ← Hero / 空态主标题
text-2xl      30px / 38px   ← 登录页大标题
text-3xl      36px / 44px   ← 营销页（不在 MVP）
```

**默认正文 14px**（开发者工具的常规密度），重要标题最大 30px。

### 2.4 字重

```text
font-normal    400   ← 正文
font-medium    500   ← 按钮、tag、表头
font-semibold  600   ← 区块标题、强调
font-bold      700   ← 大标题（少用）
```

### 2.5 间距系统（4 倍数）

```text
0    0
1    4px
2    8px
3    12px
4    16px   ← 默认 padding
5    20px
6    24px   ← 大区块间距
8    32px
10   40px
12   48px
16   64px
```

只用上述档位，不允许 5px / 7px / 13px 等非标准值。

### 2.6 圆角

```text
radius-sm    2px   ← tag / 微小元素
radius       4px   ← 按钮、input（默认）
radius-md    6px   ← 卡片、modal
radius-lg    8px   ← 容器
radius-full  9999  ← 圆形头像
```

### 2.7 阴影

```text
shadow-sm    0 1px 0 rgba(31,35,40,0.04)
shadow       0 3px 6px rgba(31,35,40,0.06), 0 1px 2px rgba(31,35,40,0.04)
shadow-md    0 8px 24px rgba(31,35,40,0.08)
shadow-lg    0 16px 48px rgba(31,35,40,0.12)
shadow-none  none
```

绝大多数页面用 `shadow-none` + `border 1px solid neutral-200`，避免阴影泛滥。modal 用 `shadow-lg`。

### 2.8 图标

- **图标库**：[Lucide Icons](https://lucide.dev) 通过 `lucide-vue-next` 引入（**任务单允许新增此一项依赖**），不混用其他库
- 默认尺寸 16px，主操作按钮 14px，导航 18px
- 颜色：跟文字色 currentColor，不单独着色

### 2.9 组件库选型

**沿用现有 Ant Design Vue ^4.0.0**（已在 [apps/web/package.json](apps/web/package.json)）。理由：
- 工具型 UI 完备：Table / Form / Modal / Dropdown 等开发量低
- 中文 i18n 默认
- 与上述视觉规范基本兼容（仅需做主题 token 覆盖）

**禁止**：
- 不准引入 Tailwind / UnoCSS（与 Ant Design 主题冲突）
- 不准引入 Element Plus / Naive UI（重复造轮子）
- 不准引入 Iconfont / Font Awesome（已有 Lucide）

### 2.10 Ant Design Vue 主题 token 覆盖

在 [apps/web/src/main.ts](apps/web/src/main.ts)（或新建 `theme.ts`）中：

```ts
import { ConfigProvider } from 'ant-design-vue'

const theme = {
  token: {
    colorPrimary: '#1F6FEB',
    colorSuccess: '#1A7F37',
    colorWarning: '#D29922',
    colorError: '#CF222E',
    colorInfo: '#0969DA',
    colorTextBase: '#1F2328',
    colorBgBase: '#FFFFFF',
    borderRadius: 4,
    borderRadiusLG: 8,
    fontFamily: '-apple-system, BlinkMacSystemFont, "Segoe UI", "PingFang SC", sans-serif',
    fontSize: 14,
  },
  components: {
    Button: { fontWeight: 500 },
    Table: { headerBg: '#F6F8FA' },
    Card: { headerBg: '#FFFFFF' },
  },
}
```

### 3.2 注册页（仅 Cloud）

**路径**：`/register`

**用户故事**：新用户邮箱注册，注册后立即创建一个 Organization。

**布局**：与登录页对称，三个字段：邮箱、密码（≥8 位，含字母数字）、再次输入密码。提交后跳转邮箱验证页 `/verify-email`。

**关键状态**：
- 邮箱已存在：字段下红字 "该邮箱已注册，去登录"
- 密码不符合规则：字段下黄字 "密码至少 8 位，包含字母和数字"
- 注册成功：跳转验证邮箱页面 "我们向 you@example.com 发送了验证链接，请查收"

## 5. 交互与动效

### 5.1 加载

- 整页：骨架屏（不用 spinner）
- 局部数据：骨架屏 / inline spinner（按钮内）
- 部署日志：流式（不显示 loading）

### 5.2 转场

- 默认无转场（即时切换）
- 折叠展开：220ms cubic-bezier(0.4, 0, 0.2, 1)
- Modal 进入 / 离开：fade 200ms

### 5.3 操作反馈

- 操作成功：toast / inline 绿字 / 状态变化（**三选一，不重复**）
- 操作失败：toast 红色 + 详细错误说明
- 网络错误：全局拦截显示 banner，提示重试

### 5.4 键盘快捷键（Phase 2+，本期不强求）

- ⌘K：全局搜索
- ⌘N：新建（按当前页上下文判断）
- ⌘.：命令面板
- /：聚焦搜索框
- Esc：关闭 Modal

## 6. 暗色模式

### 6.1 切换

- 顶栏右侧月亮/太阳图标切换
- 默认：跟随系统 `prefers-color-scheme`
- 手动选择后存 `localStorage.theme`

### 6.2 实现

- 所有颜色用 CSS 变量定义在 `:root[data-theme="light"]` 和 `:root[data-theme="dark"]`
- 不准在 component 里写硬编码十六进制
- Ant Design Vue 用 `ConfigProvider :theme` 切换

### 6.3 暗色色板映射

```text
亮: neutral-25 (页面背景)  → 暗: dark-bg
亮: neutral-0 (卡片)       → 暗: dark-surface
亮: neutral-200 (边框)     → 暗: dark-border
亮: neutral-700 (主文字)   → 暗: dark-text-1
亮: neutral-500 (次文字)   → 暗: dark-text-2
亮: brand-500              → 暗: brand-400 (略亮，对比度更高)
亮: success-500            → 暗: success-500 (保持)
```

## 7. 文案规范

### 7.1 语言

- 中文为主（按钮、菜单、提示、错误信息）
- 代码标识符 / 命令 / URL / 技术术语保持英文（commit、PR、branch、Docker、SSH 等）
- 不混用繁简，不用网络用语（"诶""啦"）

### 7.2 标点

- 中文标点用全角，但代码 / 命令保持半角
- 标题不加句号
- 按钮文本不加句号、感叹号

### 7.3 用词约定

| 不用 | 用 |
| --- | --- |
| 删除 | 删除（保留）/ 移除（轻量） |
| 您 | 你 |
| 请稍候 | 加载中… |
| 哎呀 / 哦 / 抱歉 | （直接给原因） |
| 发布成功！🎉 | 已发布 |
| 创建 / 新增 / 新建 | 创建 |
| 配置 / 设置 | 设置（个人/组织级），配置（项目/组件级） |
| 用户 | 成员（组织内）/ 用户（账号系统） |
| 工程 / 工程项目 | 项目 |
| 服务 / 模块 / 应用 | Component |
| 服务器 / 主机 / 节点 | 部署目标 |

### 7.4 错误信息

- 说清楚**发生了什么** + **怎么办**
- 例：`部署失败：SSH 连接被拒绝（host: 1.2.3.4）。请检查目标机器的 SSHD 服务是否运行，或在「部署目标」重新验证凭据。`
- 不说："出错了，请稍后再试"
- 不说："Internal Server Error"

### 7.5 空态文案

- 主标题：5-8 字，描述事实（"还没有项目"）
- 副文案：一句话，告诉用户接下来该做什么
- 按钮：3-5 字，动词开头（"创建项目""添加部署目标"）

## 8. 响应式

### 8.1 断点

```text
mobile     < 640        默认（MVP 不做移动端适配，仅保证不彻底碎）
tablet     >= 640 - 1024  侧栏自动折叠到 56px 仅图标
laptop     >= 1024 - 1440 默认布局
desktop    >= 1440        max-width 1280 居中
```

### 8.2 MVP 移动端策略

- **MVP 不支持移动端**，但页面在 < 640 时不能完全碎掉
- 顶栏 logo + 用户菜单保留，主导航变汉堡菜单
- 侧栏完全隐藏
- 表格横向滚动
- Modal 全屏
